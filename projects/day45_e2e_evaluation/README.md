# Day 45 - 仓库 Agent 端到端评测体系

这个项目用于 Day 45 的本地练习：给金融信贷离线/实时仓库 Agent 建立端到端评测集。

它不接真实 LLM、数据库或实时流，而是用固定测试集模拟当前 Agent 的表现。
今天的重点不是追求 100% 通过，而是把成功判定标准和失败归因写清楚，为 Day 46 的错误治理准备输入。

## 练习目标

- 整理 24 条覆盖真实信贷业务的数据问答问题。
- 每条问题都写清预期最终状态，例如回答、拒答、补充条件、资料不足、实时延迟或系统失败。
- 检查工具路线是否符合生产边界，尤其是离线查询必须校验 SQL，实时问题不能误走离线 SQL。
- 统计失败类型和失败业务类别，形成可修复的 bad case 清单。

## 运行方式

在仓库根目录执行：

```bash
python3 projects/day45_e2e_evaluation/main.py
```

运行后生成：

```text
projects/day45_e2e_evaluation/output/e2e_testset.json
projects/day45_e2e_evaluation/output/e2e_eval_results.json
projects/day45_e2e_evaluation/output/e2e_eval_report.md
```

## 生产映射

真实公司里的仓库 Agent 端到端评测会接入线上或准生产链路，覆盖：

- 离线指标查询：验证 Schema Catalog、SQL 生成、SQL 校验、分区裁剪、执行和解释。
- 实时指标查询：验证实时窗口、事件时间、处理时间、延迟状态和实时工具路由。
- 告警解释：验证告警证据、告警等级和禁止编造业务原因。
- 口径问答：验证 RAG 召回、指标字典引用和无依据拒答。
- 血缘追溯：验证来源表、事件流、加工任务和下游影响范围。
- 敏感请求：验证手机号、身份证号、联系人、住址等敏感信息不会泄露。
- 条件缺失：验证系统会要求补充时间、窗口、指标、产品或业务域。
- 系统异常：验证超时、分区缺失、实时延迟和依赖失败不会被模型编造成正常答案。

金融信贷场景下，评测体系的价值是让 Agent 的质量可度量、可回归、可审计。
每次改 Prompt、工具路由、SQL 校验、实时状态工具或结果解释，都应该先跑固定评测集，
再决定能不能继续推进。
