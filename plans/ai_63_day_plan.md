## 每日学习内容生成规则

从 Day 45 开始，每天新学习内容必须同时包含：

- Agent 相关知识：意图识别、工具路由、前置条件、失败回退、评测、审计、解释、服务化；
- 数据仓库相关知识：离线分层、实时链路、指标口径、Schema Catalog、血缘、权限、质量、告警。

每天固定包含以下模块：

| 模块 | 必须回答的问题 |
|------|----------------|
| 今日目标 | 今天要掌握什么能力，能放到哪个真实项目环节里？ |
| 大白话解释 | 用简单话解释核心概念，先保证能听懂。 |
| 生产实际 | 真实公司里怎么用？谁会用？接在哪个系统或链路上？ |
| 常见坑 | 线上最容易出什么问题，比如权限、性能、成本、脏数据、实时延迟、错误输出。 |
| 工程取舍 | 为什么不用更简单或更复杂的方案？怎么在准确率、成本、延迟、实时性之间取舍？ |
| 本地练习 | 今天在本地做一个最小可运行产物；代码必须按 `docs/code_commenting_guidelines.md` 写大白话注释。 |
| 面试沉淀 | 只记录生产高频问题，补充到 `docs/interview_core_questions.md`；学习内容增加时，面试沉淀也要覆盖新增核心模块。 |
| 每日核心自测 | 固定 10 个：A 区今日核心问题 7 个，B 区前三天核心回顾 3 个；Day 45-63 要同时覆盖 Agent 和数据仓库要点。 |
| 术语更新 | 新名词补充到 `notes/terminology_glossary.md`。 |

回答风格要求：

- 普通段落按 80-100 字符左右合理换行。
- 优先结合金融信贷、RAG、NL2SQL、Agent、离线仓库、实时仓库、FastAPI 服务化这些真实方向。
- 少写“我今天学了什么”，多写“生产环境为什么需要它、怎么落地、怎么排查问题”。
- 代码示例和项目代码都要补注释：不只说明怎么用，还要说明为什么要这样写、解决什么问题。
