# SQL 解释助手能检查哪些风险？

> 来源: 核心题库 | 分类: NL2SQL

SQL 解释助手可以检查语法、结构和生产执行风险。常见风险包括没有 `where` 条件、
没有 `dt` 或其他分区限制、`select *` 读取过多字段、全局 `order by` 成本高、
缺少 `limit`、大表 join 条件不清晰、字段或表名不明确、可能扫描过大数据量，
以及出现 `delete`、`update`、`drop` 等危险语句。
生产里 SQL 风险检查不能只依赖模型自然语言判断，最好把高风险规则做成确定性校验。
LLM 可以负责解释和补充建议，但是否允许执行、是否缺少分区、是否包含危险操作，
应该由规则、SQL parser、权限系统和执行层限制共同控制。这样才能减少误判，
也能把金融信贷开发经验迁移到 NL2SQL、SQL 审核和数据问答场景里。
