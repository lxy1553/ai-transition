# 被 SQL 校验拦截的问题，结果解释层应该怎么回答？

> 来源: 核心题库 | 分类: NL2SQL

被 SQL 校验拦截的问题，结果解释层不能回答成“没有数据”，也不能猜测敏感信息。
它应该明确说明该问题没有执行数据库查询，原因是权限、敏感字段、缺少时间范围、
危险 SQL 或高成本风险。
同时要给出安全替代路径，例如补充时间范围、改查汇总指标、走敏感数据审批流程，
或联系数据 owner。
这样业务用户能理解系统是在做安全控制，而不是查询失败或数据不存在。
在金融信贷场景里，这一点尤其重要，因为手机号、身份证号、客户明细和贷后敏感数据
都必须有权限和审计边界。
