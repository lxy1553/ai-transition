# 金融信贷 Agent 为什么需要指标字典，指标字典应该包含哪些字段？

> 来源: 核心题库 | 分类: NL2SQL

金融信贷 Agent 需要指标字典，是因为数据问答不能只知道表名和字段名，
还必须知道指标怎么算、按什么时间统计、能按哪些维度拆分、谁有权限查看。
如果没有指标字典，Agent 很容易混淆授信通过率、放款成功率、风控拒绝率、M1 逾期率这类指标，
生成的 SQL 可能能跑，但业务口径是错的。

一个生产级指标字典至少应该包含指标 ID、指标名称、主题域、指标类型、定义、分子、分母、
时间口径、统计粒度、来源表或来源事件、可用维度、权限等级、owner、适用范围和使用限制。
如果是实时指标，还要额外写清窗口大小、事件时间、处理时间、延迟阈值和告警规则。

在 Agent 链路里，用户问“这个指标怎么算”时，应该走指标字典或 RAG 口径问答；
用户问“这个指标当前是多少”时，才进入离线 SQL 或实时指标工具。
这样才能把口径解释、指标查询和敏感明细访问分开控制，保证回答可追溯、可审计、可回归。
