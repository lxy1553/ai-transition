# NL2SQL 查询执行层为什么必须放在 SQL 校验之后？

> 来源: 核心题库 | 分类: NL2SQL

查询执行层必须放在 SQL 校验之后，因为执行层一旦访问数据库，风险就从“生成错误”
变成了真实的数据泄露、性能冲击或误操作风险。
SQL 生成层可能产生字段越权、缺少时间范围、全表扫描、敏感字段查询或危险关键字。
这些问题必须先由 SQL Validator、权限系统和成本控制拦住。
执行层只应该接收已经通过校验的 SQL 或查询任务，使用只读连接执行，
并记录执行状态、耗时、返回行数和审计信息。
