# NL2SQL 为什么需要结果解释层？

> 来源: 核心题库 | 分类: NL2SQL

NL2SQL 需要结果解释层，是因为业务用户通常不只需要 SQL 返回的原始行列结果，
还需要理解这个结果代表什么、是否有风险、下一步该怎么追问。
查询执行层返回的是结构化数据，比如字段、行数、指标值、趋势序列和明细记录。
结果解释层要把这些内容转换成业务语言，例如说明哪个渠道通过率最高、逾期率比上周下降了多少、
某个申请编号当前审批状态是什么。
同时，解释层必须补充时间范围、指标口径、权限边界和样本限制，避免业务误读。
它不能编造 SQL 没查出来的原因，只能基于查询结果和元数据解释。
