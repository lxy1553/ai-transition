# Agent 查询离线指标时，为什么应该优先选择 ADS 或 DWS？

> 来源: 核心题库 | 分类: NL2SQL

Agent 查询离线指标时优先选择 ADS 或 DWS，是因为这两层更接近业务指标。
ADS 面向日报、看板和经营总览，指标口径已经固定；DWS 面向主题汇总和维度分析，
适合按渠道、产品、日期、风险等级做趋势和对比。

如果 Agent 默认从 DWD 或 ODS 明细临时聚合，风险会明显增加。
第一，SQL 更复杂，容易漏分区、漏过滤条件或重复计数；
第二，扫描成本更高；
第三，更容易碰到敏感字段和权限边界；
第四，指标口径可能和业务看板不一致。

所以生产级离线指标 Agent 应该先通过意图识别和分层路由判断问题粒度：
总览走 ADS，维度趋势走 DWS，授权明细才走 DWD，敏感导出直接阻断。
之后再进入 Schema Catalog、SQL 生成、SQL 校验、执行、解释和审计。
