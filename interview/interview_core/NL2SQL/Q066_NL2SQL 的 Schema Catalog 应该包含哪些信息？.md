# NL2SQL 的 Schema Catalog 应该包含哪些信息？

> 来源: 核心题库 | 分类: NL2SQL

Schema Catalog 不能只记录表名和字段名，必须包含足够信息让系统判断
"这个用户问题应该查哪张表、用哪些字段、按什么粒度聚合"。

每条记录至少包含：表名、表中文描述（这张表是干什么的）、业务主题域
（授信/风控/放款/还款/逾期）、仓库层级（ODS/DWD/DWS/ADS）、表粒度
（日级明细/渠道天汇总/日报指标）、主键、分区字段和分区格式、记录数量级。

每个字段至少包含：字段名、字段类型、字段中文业务含义（不是英文翻译）、
字段角色（指标字段/维度字段/时间字段/敏感字段/分区字段）。
指标字段要写清聚合方式（count/sum/avg/max），敏感字段要写清权限等级。
枚举字段要写清取值含义（如 status=1 是"审批通过"，不是让模型猜）。

生产来源通常来自元数据平台、Hive Metastore、数据地图或 dbt 文档。
核心价值：SQL 生成器只能从 Catalog 中选择表和字段，不能自由发明字段名。
这样既防止模型编造字段，也让选表选字段的过程可解释、可审计。
