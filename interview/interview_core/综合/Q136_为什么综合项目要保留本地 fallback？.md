# 为什么综合项目要保留本地 fallback？

> 来源: 核心题库 | 分类: 综合

因为这个项目既是学习仓库，也是作品集。本地 fallback 有三个核心价值：

第一，保证演示稳定。如果所有能力都强依赖外部模型和在线环境，演示时网络抖动或模型不可用就会导致翻车。
第二，保证回归可重复。确定性规则的结果不会因为模型版本变化而漂移，评测集每次跑出来的结果一致。
第三，把确定性规则和模型能力分开讲清楚。对于 SQL 安全、权限拦截、延迟判断这类关键逻辑，
本来就不应该完全依赖模型——它们应该走确定性代码，模型只负责理解和表达。

保留本地 fallback 也能让你在面试时说清楚"哪些用规则兜底、哪些交给模型"的工程取舍。
