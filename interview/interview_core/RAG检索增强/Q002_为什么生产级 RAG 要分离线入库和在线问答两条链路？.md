# 为什么生产级 RAG 要分离线入库和在线问答两条链路？

> 来源: 核心题库 | 分类: RAG检索增强

因为文档处理和用户问答的节奏不同。离线入库负责提前把资料清洗、切分、
向量化并建好索引；在线问答负责快速检索、拼上下文和生成答案。
这样能降低在线延迟，也方便做知识库更新、权限管理和质量评估。
离线入库链路处理的是“知识怎么进入系统”，比如解析文档、抽取正文、
清洗噪声、去重、脱敏、按标题或语义切 chunk、生成 embedding，
并把向量、原文、来源、权限标签和版本信息写入索引。在线问答链路处理的是
“用户来了怎么回答”，比如鉴权、理解问题、检索候选片段、过滤无权限内容、
重排、构造 prompt、调用 LLM 并返回引用。如果每次用户提问时才临时处理文档，
系统会非常慢，也不好控制稳定性。分成两条链路后，离线侧可以批量加工和更新知识库，
在线侧可以专注低延迟、高质量和高可用。
