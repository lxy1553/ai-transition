# 生产环境里的 RAG 系统一般怎么设计？

> 来源: 核心题库 | 分类: RAG检索增强

生产环境里的 RAG 通常不是一个简单的“向量检索 + 调模型”。
它一般会分成文档接入、解析清洗、切分、embedding、索引存储、在线检索、
权限过滤、重排、上下文构造、LLM 生成、引用返回和监控评测。
核心目标是让模型基于可信资料回答，并且答案可追溯、可控制、可优化。
我会把生产级 RAG 看成一套完整工程系统。离线侧负责知识库建设，
包括接入 PDF、Markdown、网页、数据库说明、指标文档等资料，然后做解析、清洗、
去重、脱敏、切分、embedding 和索引构建。在线侧负责用户问答，包括鉴权、
问题理解、query rewrite、向量检索、关键词检索、权限过滤、rerank、上下文拼装、
调用 LLM 生成答案，并返回引用来源。生产环境还要考虑监控和评测，
比如召回是否命中、回答是否基于资料、无答案率、延迟、成本和用户反馈。
所以 RAG 的难点不只是模型，而是资料质量、检索质量、权限安全和工程稳定性。
