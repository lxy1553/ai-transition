# 调用外部 API 或 LLM API 时要怎么保证稳定性？

> 来源: 核心题库 | 分类: 工程化

要设置 timeout、重试、限流、熔断、错误分类和日志。
还要处理 4xx、5xx、网络超时、返回格式异常和额度不足。
生产里不能假设外部 API 永远成功，要有失败兜底和可观测性。
调用外部 API 时，第一步是设置合理 timeout，避免请求一直卡住。
第二是做有限重试，通常只对网络抖动、超时、部分 5xx 做重试，
不要对参数错误或权限错误盲目重试。第三是错误分类，比如 401 是鉴权问题，
429 是限流问题，5xx 是服务端问题，JSON 解析失败是响应格式问题。
第四是日志和指标，记录请求耗时、状态码、错误类型和重试次数。
对于 LLM API，还要关注 token 成本、上下文长度、模型不可用和输出格式异常。
必要时要有降级策略，比如返回规则结果、提示稍后重试，或者切换备用模型。
