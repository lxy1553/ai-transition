# AI API 服务为什么需要配置、日志和异常处理？

> 来源: 核心题库 | 分类: 工程化

配置解决环境差异和密钥管理，日志解决排查和审计，异常处理解决服务稳定性。
AI 服务会调用模型、向量库、数据库和外部 API，出错点很多。
没有这些工程能力，服务很难稳定上线。
生产 AI API 服务不是只要本地能跑通。配置管理让不同环境使用不同参数，
比如模型名、API Key、数据库地址、向量库地址、timeout 和日志级别。
密钥不能写死在代码里，要通过环境变量或密钥管理系统注入。
日志用于排查问题和做审计，比如记录请求 ID、用户、耗时、调用链路、错误码和关键结果。
异常处理用于把底层错误转换成稳定响应，避免把堆栈、密钥或内部路径暴露给用户。
RAG 和 NL2SQL 的链路长，任何一个环节失败都要有清晰错误和降级策略。
这些能力决定服务能不能从脚本变成线上系统。
