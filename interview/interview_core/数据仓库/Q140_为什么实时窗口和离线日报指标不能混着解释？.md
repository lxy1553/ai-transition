# 为什么实时窗口和离线日报指标不能混着解释？

> 来源: 核心题库 | 分类: 数据仓库

因为框架只是实现手段，业务价值来自数据边界、工具路由、安全控制、
审计和评测。

如果开口就是"我用 LangChain 搭了个 Agent"，面试官听到的是：
你关注的是用什么工具，而不是解决了什么问题。

正确的讲法是：先用业务问题开头（金融信贷场景为什么需要同时处理
离线指标和实时告警），再讲系统如何通过意图识别、工具路由、
SQL 校验和审计来保证安全和可控。框架只是"我们在哪些环节用它
加速开发"的补充说明，出现在最后。

比如："在工具注册和编排层我们用了 LangGraph 的 StateGraph
来管理多步调用和状态流转，因为它的状态管理和节点跳转机制
比较适合我们 topic_router → intent_router → tool_executor
的链式编排。但对于权限校验、SQL Validator 和审计落库这些
确定性逻辑，我们是代码层独立实现的，不依赖任何框架。"

一句话：框架是手段，不是项目价值本身。
面试官更关心你解决了什么问题、做了哪些工程取舍。
