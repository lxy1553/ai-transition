# 生产级 AI 应用为什么需要端到端评测，而不是只测单个 Prompt 或工具？

> 来源: 核心题库 | 分类: Agent

生产级 AI 应用需要端到端评测，因为真实用户关心的是完整链路结果，
而不是某个 Prompt、检索器、SQL 生成器或实时工具单独表现。
一个金融信贷仓库 Agent 可能每个单点工具都能跑通，但整体仍然出错。
例如实时问题误走离线 SQL、缺少时间范围却继续查库、SQL 生成后跳过校验、
指标口径没有引用却回答、实时链路延迟却仍给出确定结论，都会导致生产风险。

端到端评测应该从用户问题开始，检查意图识别、上下文检索、工具路由、离线 SQL 生成、
SQL 校验、离线查询执行、实时指标查询、告警解释、口径引用、血缘追溯、安全阻断和审计记录。
每条样例都要有明确成功判定，包括预期最终状态、必须调用的工具、禁止调用的工具、
是否需要引用、是否允许执行 SQL、是否需要检查实时延迟和失败分类。
对敏感导出、越权访问、资料不足、实时延迟和系统异常来说，正确拒答、正确澄清或正确降级也应该算成功。

在金融信贷场景里，端到端评测尤其重要，因为系统可能接触授信、风控、放款、还款、
逾期、催收、实时反欺诈和客户敏感信息。
固定评测集可以持续回归，帮助团队判断每次 Prompt、规则、Schema Catalog、指标字典、
实时指标字典、血缘工具或工具路由改动是否真的提升质量，是否引入新的安全、合规或实时风险。
