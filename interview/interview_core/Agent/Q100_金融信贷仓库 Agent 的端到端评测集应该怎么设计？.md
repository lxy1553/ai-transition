# 金融信贷仓库 Agent 的端到端评测集应该怎么设计？

> 来源: 核心题库 | 分类: Agent

金融信贷仓库 Agent 的端到端评测集不能只放“问指标并回答”的成功样例，
而要覆盖离线指标、实时指标、告警解释、指标口径、血缘追溯、权限阻断、缺参澄清和系统异常。
每条样例都应该从真实用户问题出发，写清业务类别、预期状态、必须调用的工具、禁止调用的工具、
是否需要引用、是否允许执行 SQL、是否要检查实时延迟，以及失败后应该归到哪类责任层。

离线指标样例要检查 Schema Catalog、分区字段、指标口径、SQL 生成、SQL 校验、执行和结果解释。
例如查询“昨日各渠道授信通过率”，必须走离线指标链路，不能查明细敏感字段，
也不能跳过 SQL Validator。
实时指标样例要检查窗口、事件时间、处理时间、延迟状态和实时工具路由，
例如“近 5 分钟实时拒绝率是否异常”不能用离线日报表回答。

评测集还必须包含应该失败的问题。
例如导出客户手机号、查询无权限明细、缺少时间范围、口径资料没有引用、分区缺失、实时链路延迟和数据库超时。
这些样例的正确结果不是硬答，而是安全阻断、要求补充条件、资料不足或系统异常兜底。
这样评测集才能真正保护生产边界，而不是只证明 Demo 能跑。
