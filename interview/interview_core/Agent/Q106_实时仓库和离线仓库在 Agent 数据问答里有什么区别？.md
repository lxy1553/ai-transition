# 实时仓库和离线仓库在 Agent 数据问答里有什么区别？

> 来源: 核心题库 | 分类: Agent

离线仓库面向稳定的历史指标和经营分析，通常按天或批次加工数据。
Agent 查询昨日、上周、上月的授信通过率、放款金额、逾期率时，应该走离线 ADS 或 DWS。
实时仓库面向当前状态、分钟级窗口、实时特征和告警，适合回答近 5 分钟申请量、当前拒绝率是否异常、
实时还款失败数是否突增这类问题。

两者最大的区别不是表名不同，而是时间语义不同。
离线指标强调分区、批次、指标口径和稳定性；实时指标强调事件时间、处理时间、窗口、延迟和链路状态。
如果 Agent 把实时问题路由到离线 SQL，即使能查出数字，也可能业务上错误。

生产级 Agent 应该先识别问题是离线指标还是实时状态。
离线问题走 Schema Catalog、SQL 生成、SQL 校验和离线执行；
实时问题走实时指标工具或告警工具，并检查窗口和延迟状态。
敏感明细和实时事件流导出都必须经过权限控制和审计。
