# 为什么金融信贷 Agent 项目不能只强调用了 LangChain / LangGraph / LangSmith？

> 来源: 核心题库 | 分类: Agent

因为框架只是实现手段，业务价值来自数据边界、工具路由、安全控制、审计和评测。

如果只强调框架名，面试官会认为你是在堆技术词，而不是解决真实场景问题。
正确的讲法是：先用业务问题开头（金融信贷场景为什么需要同时处理离线指标和实时告警），
再讲系统如何通过意图识别、工具路由、SQL 校验和审计来保证安全和可控，
框架只是"我们在哪些环节用它加速开发"的补充说明。

一句话：框架是手段，不是项目价值本身。面试官更关心你解决了什么问题、做了哪些取舍。
