# LLM Agent 常⻅功能有哪些？

> 来源: 面试鸭题库 | 分类: Agent 与框架

LLM Agent 不是简单的问答机器⼈，它得能感知环境、做规划、调⼯具、持续执⾏直到⽬标完成。你可以把它看成⼀
个“AI 执⾏官”，⾃⼰拆任务、调接⼝、纠错重试。
1）规划与⽬标分解
Agent 接到“分析竞品并写报告”这种⼤任务，不会⼀⼝吞，⽽是拆成查资料、读 PDF、对⽐数据、⽣成初稿、润⾊
⼏步。像 LangChain 的 PlanAndExecute  就是⼲这个的典型模式。
2）⼯具调⽤（Function Calling）
它得知道什么时候该让别⼈⼲活。⽐如查实时股价，直接调 Alpha Vantage API；订会议室就⾛钉钉或⻜书开放平台。
模型输出结构化指令，框架负责转成真实调⽤。
3）记忆能⼒
短时记忆靠上下⽂窗⼝存对话历史，⻓时记忆就得接外部存储。⽐如⽤ Chroma 或 Milvus 存过往决策记录，下次遇到
类似任务能参考。
4）⾃主执⾏与反思
有些 Agent 能⾃⼰决定下⼀步。AutoGPT 是个例⼦，但它容易陷⼊循环。更稳的做法是 ReAct 模式：每步“思考-⾏
动-观察”，像⼈⼀样边做边看结果。
5）多 Agent 协作（进阶）
复杂场景下会分⼯。⽐如⼀个当产品经理提需求，⼀个当⼯程师写代码，⼀个当测试员验证，通过内部通信达成⽬
标。MetaGPT 就模拟了这种⻆⾊分⼯。
别指望它百分百靠谱，超⻓上下⽂推理还是会乱，⼯具调⽤也可能失败。关键是设计好 fallback 机制和⼈⼯介⼊点。
