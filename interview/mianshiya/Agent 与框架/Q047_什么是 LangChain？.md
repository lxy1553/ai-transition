# 什么是 LangChain？

> 来源: 面试鸭题库 | 分类: Agent 与框架

LangChain 是个帮助开发者快速搭建应⽤的框架，重点是让⼤模型和外部系统打通。你不⽤从零写 prompt 调⽤、上
下⽂管理、⼯具调⽤这些重复代码，它都给你封装好了。
它的核⼼能⼒之⼀是链式调⽤，你可以把多个步骤串起来，⽐如先查数据库，再让⼤模型总结，最后发个通知。每个
环节它都抽象成组件，像积⽊⼀样拼接。
另⼀个关键点是跟外部数据对接。⽐如你要做个客服机器⼈，不能光靠模型内置知识，得让它查你的产品⽂档。
LangChain 提供了和向量数据库集成的能⼒，像和 Pinecone、Weaviate 对接，做检索增强⽣成（RAG），这样回答就
有据可依。
它还⽀持 Agent 模式，就是让模型⾃⼰决定下⼀步⼲什么。⽐如⽤户问“昨天销售额最⾼的订单是谁”，模型可以⾃⼰
判断要先查订单表，再查客户信息，然后组织语⾔回复。背后其实是模型输出特定格式指令，框架负责解析并执⾏。
代码上它很简洁，⼏⾏就能定义⼀个链：
from langchain.chains import LLMChain
chain = LLMChain(llm=llm, prompt=prompt)
response = chain.run(" 中国的⾸都是哪⾥？ ")
适合做需要动态输⼊、多步推理、结合私有数据的场景，⽐如智能客服、数据分析助⼿。但如果你只是简单问答，可
能显得重了。
