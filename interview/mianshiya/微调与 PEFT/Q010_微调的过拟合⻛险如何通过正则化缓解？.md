# 微调的过拟合⻛险如何通过正则化缓解？

> 来源: 面试鸭题库 | 分类: 微调与 PEFT

微调时模型已经⻅过⼤量训练数据，参数调整幅度⼩但密集，容易记住训练集的特定模式，导致在新数据上表现下
滑。正则化的核⼼作⽤就是在损失函数或训练过程中加⼊约束，让模型别学得太“死”。
1）L2 正则（权重衰减）是最常⽤的⼿段。它在损失函数⾥加上所有权重的平⽅和，迫使参数值趋向更⼩。⽐如
Hugging Face 的 Trainer  ⾥设置 weight_decay=0.01 ，就能防⽌某些权重变得过⼤，提升泛化能⼒。
training_args = TrainingArguments(
output_dir="output",
weight_decay=0.01
)
2）Dropout 在微调阶段依然有效。虽然预训练模型内部已经固化了结构，但在分类头甚⾄最后⼏层中间加 Dropout，
能让输出不依赖于少数神经元。⼀般保留率设在 0.1〜0.3 就够了。
3）早停（Early Stopping）特别适合微调场景。因为微调轮次少，可能只跑 3〜5 个 epoch，验证集性能⼀旦下降就
终⽌，能避免多⾛⼀两步掉进过拟合坑⾥。
4）还有像 标签平滑（Label Smoothing）这种软化监督信号的⽅法。它不让模型对真实标签过于⾃信，间接抑制过拟
合。在分类任务中设 label_smoothing_factor=0.1  是常⻅做法。
这些⽅法本质都是给学习过程加点“不确定性”或“惩罚”，让模型别把微调当背书，稍微收着点劲。
