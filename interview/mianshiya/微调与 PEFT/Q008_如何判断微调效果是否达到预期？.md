# 如何判断微调效果是否达到预期？

> 来源: 面试鸭题库 | 分类: 微调与 PEFT

微调效果能不能扛住业务，得靠数据说话，不能拍脑袋。
1）先看基础指标，⽐如分类任务的准确率、F1 值，⽣成任务的 BLEU、ROUGE。这些是硬指标，训练集和验证集上
的提升得稳定，⼀般要求验证集指标⽐基线模型⾼ 5% 以上才算有显著收益。
2）光看数字不够，得做⼈⼯评估。抽样⼀批模型输出，让业务⽅或标注⼈员打分，重点关注⽣成内容是否符合语义、
逻辑通顺、不胡说⼋道。像⽤ Qwen 或 Llama 微调后，常出现“套话连篇但答⾮所问”的问题，这种机器指标发现不
了。
3）上线前必须⾛ A/B 测试。把⽼模型和新模型同时跑在线上，按流量切分，对⽐点击率、⽤户停留时⻓、转化率这些
业务指标。⽐如在客服场景⽤微调模型，⽬标是降低⼈⼯转接率，那这个值下降 10% 以上才叫真有效。
4）还得看泛化性。拿⼀些边缘 case 或分布外数据测试，⽐如新增的⽤户 query 类型，模型能不能兜住。如果⼀遇到
没⻅过的句式就崩，说明过拟合了，得回炉加数据增强或者正则。
代码层⾯可以写个简单的评估脚本：
from sklearn.metrics import f1_score
f1 = f1_score(y_true, y_pred, average='weighted')
print(f" 微调后 F1: {f1:.3f}")
最后记住，过拟合是微调最⼤的坑，训完训练集接近 100% 准确率，验证集却卡在 70%，基本就是背答案了。数据质
量差、样本太少、学习率太⾼都可能引发这问题。
介绍⼏种常⻅的微调策略的优缺点
微调⼤模型不是上来就全量参数⼀把梭，得看资源和场景做选择。
1）全量微调把所有参数放开训练，效果最好，像 Llama 系列在特定任务上刷榜基本都靠它。但代价也⼤，⼀张 A100
训个 7B 模型就得跑好⼏天，显存轻松⼲到 80G 以上，⼩团队根本搞不定。
2）LoRA（低秩适配） 是现在最⽕的轻量⽅案。它冻结原模型，在输⼊层加两个低秩矩阵做增量，训练时只更新这两
块。⽐如 HuggingFace 上很多基于 Qwen、ChatGLM 的衍⽣模型都是⽤ LoRA 微出来的。显存能压到 20G 内，训练
快，还能多任务共⽤⼀个底模型。缺点是超参敏感，r 秩选不好容易⽋拟合。
3）Adapter Tuning 在 Transformer 层中间插⼊⼩型神经⽹络模块，训练时只动这些⼩模块。结构改动⽐ LoRA ⼤，
推理有延迟，但现在基本被 LoRA 吊打，除⾮你对模块化设计有强需求。
4）Preﬁx Tuning 和 Prompt Tuning 都是往输⼊前⾯塞可学习的向量。前者改 attention key/value，后者直接拼接
embedding。适合少样本场景，但⻓序列下效果不稳定，⼀般只⽤于探索性实验。
⼀般来说，LoRA 是平衡效果和成本的⾸选，真要拼上限再考虑全量。
