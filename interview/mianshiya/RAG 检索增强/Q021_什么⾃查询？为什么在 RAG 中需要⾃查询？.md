# 什么⾃查询？为什么在 RAG 中需要⾃查询？

> 来源: 面试鸭题库 | 分类: RAG 检索增强

⾃查询其实是让模型⾃⼰⽣成查询条件，去检索它需要的信息。⽐如你问“苹果最新财报怎么样”，模型不会直接回
答，⽽是先拆解问题，⽣成像“苹果 2024 年 Q2 营收、利润、同⽐增⻓”这样的检索关键词，再拿这些词去查⽂档。
这在 RAG ⾥特别关键，因为原始问题往往太笼统或包含多个维度。直接拿原问题去搜，召回的内容可能不精准。⾃查
询相当于加了⼀层智能路由，把复杂问题翻译成适合检索的结构化查询语句。
1）能处理时间范围过滤，⽐如“去年销售额最⾼的产品”会被转成带时间约束的查询
2）⽀持属性筛选，“便宜的安卓⼿机”会拆成“价格低于 2000 元、操作系统为 Android”
3）适配向量库的元数据检索，⽐如⽤ Chroma 或 Pinecone 的 where 条件过滤
代码上⼀般⽤ LLM 接⼀个提⽰模板：
String prompt = " 根据⽤户问题⽣成检索查询语句 : " + userQuestion;
List<String> queries = llm.generate(prompt);
List<Document> results = vectorStore.search(queries);
整个流程像是给 RAG 装了个“理解⼤脑”，先把⼈话转化成机器能⾼效检索的指令，避免压根不经过语义解析就硬
搜。
