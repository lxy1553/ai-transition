# 什么是 Modular RAG？

> 来源: 面试鸭题库 | 分类: RAG 检索增强

Modular RAG 不是某个固定框架，⽽是指⼀种将 RAG（检索增强⽣成）系统拆解成独⽴、可替换模块的设计思路。你
完全可以把它看作 RAG 的“微服务化”⸺每个环节单独优化、灵活组合。
整个流程通常包括⼏个关键阶段：⾸先从外部知识库中检索相关⽂档，然后对检索结果做重排序（Rerank），最后把处
理好的上下⽂喂给⼤模型⽣成答案。传统 RAG 把这些步骤耦合在⼀起，⽽ Modular RAG 强调每个部分都能独⽴迭
代。
⽐如你在⽤ LangChain 或 LlamaIndex 时，可以⾃由替换不同的检索器（如从 BM25 切到向量检索）、换上 Cross-
Encoder 做精排、再接⼊不同的 prompt ⼯程策略。这种结构下，⼀个团队可以专注优化检索质量，另⼀个团队负责
提升⽣成流畅度，互不⼲扰。
1）检索模块：⽀持多路召回，⽐如同时⾛关键词和语义向量 2）重排序模块：⽤更精细的模型对 Top-K 结果打分，提
升相关性 3）上下⽂融合：对多个⽚段做去重、摘要或拼接，减少噪声 4）⽣成控制：动态决定是否启⽤检索，避
免“硬套答案”
这种⽅式特别适合复杂场景，像在企业知识库中查政策，可能需要结合时间过滤、权限判断等额外逻辑，直接塞进端
到端 RAG 很难维护。拆成模块后，脏活累活交给对应组件，主链路反⽽更清晰。
Computer Use 是什么？说说它的原理
⽬前没有明确的技术标准或⼴泛认知的系统叫做 "Computer Use"。这个术语在主流计算机科学、软件⼯程或IT基础设
施领域并不指代某个具体的技术、协议或架构。
如果你指的是“计算机的使⽤”（the use of computers），那涵盖范围⾮常⼴，包括操作系统调度、程序执⾏、内存管
理、⽹络通信等基础⾏为。但作为⼀个专有名词，“Computer Use”并不是像 Redis、Kafka 或 JVM 这样的具体技术
组件。
有可能是提问时表述有误，⽐如本意想问的是：
Compute Unit：计算单元，常⻅于分布式计算框架（如 Spark 中的 executor）或硬件架构中。

或者某个特定产品/项⽬中的内部命名，例如某系统的“ComputerUseService”类。
