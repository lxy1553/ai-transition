# RAG 的完整流程是怎么样的？

> 来源: 面试鸭题库 | 分类: RAG 检索增强

RAG 的流程其实就三步：准备数据、查资料、⽣成答案。整个过程让⼤模型在回答前，先去“翻书”找依据。
1）数据准备阶段，先把外部知识库的⽂档切⽚，⽐如把 PDF 或数据库表结构说明拆成⼀段段⽂本块。接着⽤
embedding 模型给每⼀块转成向量，存进向量数据库，像 Milvus 或 Pinecone 这类⼯具⼲的就是这事。
2）⽤户提问时，系统不会直接扔给⼤模型。⽽是先把问题也转成向量，去向量库⾥做相似度搜索，找出最相关的⼏个
⽂本块。这⼀步很关键，决定了模型能“看到”哪些背景知识。
3）最后，把这些检索到的⽂本块拼接成提⽰词，连同原问题⼀起喂给⼤模型。模型基于这些上下⽂⽣成答案，⽽不是
靠记忆瞎编。这样出来的回答有据可查，幻觉少。
这个流程⾥，embedding 模型和向量检索的质量直接决定效果。如果检索错了，后⾯再强的模型也搞不定。常⻅坑是
⽂本分块不合理，⽐如按固定字符切，把⼀个完整概念切碎了，导致查不准。
