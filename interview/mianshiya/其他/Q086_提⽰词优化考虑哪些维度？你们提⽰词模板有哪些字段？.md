# 提⽰词优化考虑哪些维度？你们提⽰词模板有哪些字段？

> 来源: 面试鸭题库 | 分类: 其他

这问题其实是在问提⽰词⼯程的系统化设计思路，不只是随便写句话。
1）⾸先看维度。⼀个有效的提⽰词必须明确任务⽬标、输⼊约束和输出格式。任务类型决定结构，⽐如分类、⽣成、
抽取需要的模板完全不同。上下⽂信息也得给⾜，模型才能理解语义边界。还要考虑安全合规要求，像敏感词过滤、
价值观对⻬这些都得前置控制。
2）再说模板字段。我们常⽤的提⽰词模板包含这⼏个关键部分：⻆⾊定义，说明这个 prompt 是给谁⽤的；背景描
述，交代清楚业务场景；指令主体，这是核⼼，告诉模型具体要做什么；⽰例样本，few-shot 的例⼦能显著提升准确
率；输出约束，⽐如 JSON 格式、字数限制、枚举值范围；最后是异常处理机制，定义模糊输⼊时的降级策略。
代码⽰例⻓这样：
prompt = """
⻆⾊：电商客服助⼿
背景：⽤户在商品详情⻚发起咨询
指令：根据商品信息和⽤户问题，⽣成专业且友好的回复
输⼊：
- 商品标题： {title}
- ⽤户问题： {query}
输出格式： JSON ，包含  reply 字段，不超过  150 字
"""
这种结构能让⼤模型稳定输出符合预期的结果，⽽不是靠运⽓。
