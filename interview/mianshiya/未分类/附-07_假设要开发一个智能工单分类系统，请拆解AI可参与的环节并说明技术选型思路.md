# 假设要开发一个智能工单分类系统，请拆解AI可参与的环节并说明技术选型思路

> 来源: 面试鸭题库 | 分类: 未分类

智能⼯单分类不是简单打个标签，得把AI嵌进整个流转链条⾥，哪⾥能省⼈⼒、提准确率就往哪插。
1）⼯单预处理阶段，⽂本清洗和标准化可以⽤规则+轻量模型。⽐如⽤户输⼊“登不录”，先⽤正则和纠错词典做基础
清洗，再上⼀个 TinyBERT 做语义对⻬，转成标准表述“⽆法登录系统”。这步别上⼤模型，响应延迟扛不住。
2）核⼼分类环节看数据量和类⽬复杂度。如果只有⼏⼗个固定类别，RoBERTa + CRF ⾜够，推理快、好维护。要是类
⽬动态增⻓，⽐如每天新增业务线，就得考虑 Prompt Learning 配合 P-tuning v2，接通⼤模型如 ChatGLM-6B 做
zero-shot 分类，类⽬⼀改prompt跟着变，不⽤重新训练。
3）后置反馈闭环必须做。分错的⼯单要进⼈⼯复核池，这部分数据定期回流做增量训练。可以搭个轻量 Label Studio
+ Milvus 相似样本检索，⾃动推荐历史相似案例给标注员，效率翻倍。
4）⼯程部署上，⼩模型⾛ TensorFlow Lite 或 ONNX Runtime 嵌⼊服务内部，⼤模型⽤ vLLM 做批处理异步推理。别
把所有请求都怼到⼤模型上，成本直接炸。
