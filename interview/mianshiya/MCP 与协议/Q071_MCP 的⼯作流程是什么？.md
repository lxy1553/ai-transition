# MCP 的⼯作流程是什么？

> 来源: 面试鸭题库 | 分类: MCP 与协议

MCP 其实是 Model Context Protocol 的缩写，它不是传统中间件或 Java 领域的标准组件，⽽是在 AI Agent 架构中⽤
来规范⼤模型与外部⼯具、数据源之间交互的协议。它的核⼼作⽤是让模型能安全、可控地访问运⾏时上下⽂。
整个流程从⽤户发起请求开始。Agent 接收到输⼊后，会先做⼀次意图解析，判断是否需要调⽤⼯具。如果需要，就
会构造⼀个符合 MCP 格式的请求包，⾥⾯包含动作类型、参数和上下⽂元信息。
接下来这个请求会被转发给对应的⼯具适配器。⽐如你要查数据库，就会⾛ SQL ⼯具链；要发 HTTP 请求，就交给
API 客户端处理。执⾏结果原路返回，由 MCP 解析器把原始响应转成模型能理解的⾃然语⾔⽚段。
1）请求触发：⽤户输⼊进⼊ Agent 循环
2）决策判断：Controller 判断是否需外部调⽤
3）协议封装：按 MCP 标准⽣成 tool call 消息
4）⼯具执⾏：Adapter 调⽤具体服务并获取结果
5）上下⽂注⼊：将结果以结构化⽅式回填到 prompt 上下⽂
这种设计的好处是解耦了模型逻辑和外部系统。像 LangChain 或 Semantic Kernel 这类框架，底层其实就是靠类似
MCP 的机制在做调度。你不需要改模型代码，只要注册新⼯具就能扩展能⼒。
