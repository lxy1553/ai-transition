# A2A 协议 与 MCP 协议的关系是怎样的？

> 来源: 面试鸭题库 | 分类: MCP 与协议

A2A 和 MCP 都是服务间通信的协议⽅案，但定位和层级不⼀样。
A2A ⼀般指 Application-to-Application，是业务应⽤之间的直接调⽤协议，⽐如通过 Dubbo、Spring Cloud RPC 或
gRPC 实现。这类调⽤压根不经过⻚⾯层，都是接⼝对接⼝，常⻅于微服务架构⾥订单系统调库存系统这种场景。
MCP 全称可能是 Module Communication Protocol，但在阿⾥系中间件语境下，它更多指⼀种模块级通信规范，通
常运⾏在同⼀个应⽤内部的不同逻辑模块之间，或者是轻量级跨进程通信的封装。它的抽象层级⽐ A2A 低⼀些，更像
是中间件内部组件交互的语⾔。
你可以把 A2A 看作城市之间的⾼速公路，两个独⽴城市（应⽤）之间跑整⻋货（请求体）。⽽ MCP 更像是⼀个⼯⼚内
部的传送带，不同⻋间（模块）之间传递零件，节奏更紧凑，协议更定制化。
所以两者不是替代关系，⽽是分层协作。A2A 解决的是系统边界的调⽤问题，MCP 处理的是内部模块间的协同，脏活
累活分⼯明确。实际在 SOFAStack 或类似架构中，你会同时看到这两种协议在跑。
