# 什么是 A2A 协议，它的核⼼架构及主要组件有哪些？

> 来源: 面试鸭题库 | 分类: MCP 与协议

A2A（Application-to-Application）协议不是某个具体的技术标准，⽽是描述系统间通过预定义接⼝直接通信的模
式。这类交互通常在服务之间完成数据交换或功能调⽤，不依赖⽤户参与。
⼀般架构下，A2A 的核⼼是去中⼼化或轻量⽹关控制的服务直连。常⻅于微服务、B2B 集成或内部系统协同场景。⽐
如订单系统⾃动触发库存扣减，就是典型的 A2A 调⽤。
典型组件包括：
1）服务提供⽅：暴露 RESTful API 或 gRPC 接⼝，⽐如⽤ Spring Boot 实现的⽤户中⼼服务。 2）服务消费⽅：主动
发起请求的应⽤，例如交易系统调⽤⻛控服务做决策。 3）认证与授权模块：常⽤ OAuth2 的 Client Credentials 模
式，确保机器⾝份可信。API 密钥也常⽤于轻量场景。 4）API ⽹关（可选）：像 Kong 或 Apigee，负责限流、⽇志、
路由，但⾼并发内⽹调⽤时常绕过⽹关直连以降低延迟。 5）服务注册与发现：在动态环境中，Eureka 或 Nacos 帮助
消费⽅找到可⽤实例。
代码上，⼀个简单的 A2A 调⽤可能⻓这样：
// 使⽤  WebClient 发起异步请求
webClient.get()
.uri("http://user-service/users/123")
.retrieve()
.bodyToMono(User.class)
.block();
注意点在于⽹络超时、重试策略和熔断机制。如果搞不定容错，雪崩效应会迅速扩散。因此，配合 Hystrix 或
Resilience4j 是常规操作。
