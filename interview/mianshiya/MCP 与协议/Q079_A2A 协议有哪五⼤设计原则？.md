# A2A 协议有哪五⼤设计原则？

> 来源: 面试鸭题库 | 分类: MCP 与协议

A2A（Application-to-Application）协议本质是服务间通信的契约设计，重点在于稳定、⾼效、可维护。它的五⼤设
计原则其实围绕的是“解耦”和“可控”。
1）接⼝版本化
接⼝⼀旦发布就不能随便改，否则上游调⽤⽅会炸。⼀般通过 URL 路径或 Header 携带版本信息，⽐如 /v1/order
和 /v2/order  并⾏存在，⽼接⼝逐步下线。
2）明确的契约定义
⽤ OpenAPI（Swagger）、Protobuf 或 Thrift 明确定义输⼊输出结构，字段类型、是否必填、枚举值都要说清楚。前
后端或跨团队协作时，契约先⾏，避免“我猜你返回的是个字符串”。
3）错误码标准化
别随便返回 200 包着⼀个 {code: 500}  的体。HTTP 状态码要合理使⽤，业务错误码要统⼀规范，⽐如
ORDER_NOT_FOUND=10001 ，⽂档可查，⽇志可追踪。
4）限流与熔断机制内建
系统扛不住的时候得⾃我保护。A2A 调⽤必须内置限流（如令牌桶）和熔断（如 Hystrix、Sentinel），防⽌雪崩。⽐如
下单服务被查询服务拖垮，就得把⾮核⼼链路熔断掉。
5）可观测性⽀持
每⼀次调⽤都得能追踪，通过链路追踪系统（如 SkyWalking、Zipkin）记录请求路径，结合⽇志和监控指标（延迟、
QPS），快速定位问题在哪⼀环。
