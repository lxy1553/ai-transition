# MCP 协议⽀持哪两种模式？

> 来源: 面试鸭题库 | 分类: MCP 与协议

MCP（Model Context Protocol）⽬前主要⽀持两种交互模式，对应不同的上下⽂处理⽅式。
1）请求-响应模式
这是最常⽤的模式，客户端发送⼀个请求，携带特定上下⽂，服务端完全基于该上下⽂⽣成响应，交互结束。整个过
程类似 HTTP 的调⽤模型，适合⼀次性任务，⽐如让⼤模型总结⼀段⽂本、翻译⼀句话。这种模式下，上下⽂只在当
前请求中⽣效，不会保留。
2）会话模式
系统会维护⼀个持续的上下⽂会话状态，客户端可以连续发送多条消息，每条都基于之前的对话历史进⾏理解与回
应。典型场景是聊天机器⼈、AI 助⼿这类需要记忆上下⽂的产品，⽐如使⽤ MCP 协议对接 LLM 构建客服系统时，⽤
户前⼏轮提到的信息要能在后续对话中被引⽤。
会话模式的关键在于服务端需要管理 Session 状态，可能结合 UUID 或 token 来标识不同⽤户的上下⽂隔离。⽽请求-
响应模式⽆状态，更容易⽔平扩展。
要不要⽤会话模式，得看业务是否需要“记住”前⾯的内容。如果只是做离散任务，⽐如内容审核、关键词提取，压
根不需要维持状态，⽤第⼀种就够了。
