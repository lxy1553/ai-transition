# MCP 架构包含哪些核⼼组件？

> 来源: 面试鸭题库 | 分类: MCP 与协议

MCP 架构这个词在主流技术领域没有统⼀标准定义，更常⻅的类似缩写是 MCP（Model-View-Presenter）属于 UI 架
构模式，常⽤于客户端开发。但如果你指的是服务端或中间件场景下的某种定制架构，可能是特定团队对 消息驱动、
控制平⾯分离 类系统的简称。
不过从实际系统设计⻆度看，如果把 MCP 理解为 Message-Driven Control Plane 这类架构，它的核⼼组件⼀般包含
三个部分：
1）消息中枢
负责路由和分发指令，⽐如⽤ Kafka 或 RocketMQ 做命令总线，所有控制指令通过 Topic ⼴播。消费者按需订阅，实
现解耦。这种设计在微服务治理中很常⻅，像 Nacos 配置变更就是靠⻓轮询 + 消息通知双机制推到客户端。
2）控制单元
接收消息后执⾏具体逻辑，⽐如修改本地状态、调⽤外部接⼝、触发任务调度。它不直接暴露 HTTP 接⼝，⽽是被动
响应事件。Spring Cloud Stream 的 @StreamListener 就是典型写法，压根不经过 Web 层。
3）状态反馈器
反向上报执⾏结果，形成闭环。通常往⼀个集中式存储写状态，⽐如把节点健康度打到 Redis，或者把执⾏⽇志发到
Elasticsearch。ZooKeeper 临时节点也⼲这事，会话⼀断⾃动清理状态。
