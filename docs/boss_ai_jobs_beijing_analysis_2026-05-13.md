# Boss 直聘北京 AI 方向岗位分析

> 日期：2026-05-13

> 基准位置：回龙观东大街

> 数据口径：公开可访问的 Boss 直聘索引页和岗位摘要样本，非登录后的实时全量数据。

---

## 1. 样本说明

本次主要观察北京地区以下岗位：

- AI 应用工程师
- AI Agent 开发工程师
- RAG / 大模型应用工程师
- AI 后端 / 全栈开发
- AI 搜索 / RAG 架构
- AI 平台开发
- 算法应用工程师
- AI 产品数据分析

Boss 直聘很多岗位详情需要登录或会做反爬限制，所以这里是公开索引样本统计，
适合判断市场方向、薪资区间、技能要求和投递优先级，不等同于实时可投岗位清单。

薪资统一按月薪 `K/月` 观察；如果岗位写了 `14薪 / 15薪 / 16薪 / 20薪`，
说明年包会明显高于 12 个月口径。

---

## 2. 总体结论

北京 AI 岗位目前明显分成四类：

| 类型 | 典型薪资 | 适合程度 | 说明 |
|------|----------|----------|------|
| AI 应用 / RAG / Agent 后端 | 20-40K、30-60K | 高 | 最贴合当前学习路线，重工程落地 |
| AI 搜索 / RAG 架构 | 40-60K、50-70K | 中高 | 薪资高，但更要求架构和经验 |
| 大模型算法 / 多模态 / 训练 | 30-60K、50-80K | 中 | 更偏算法、硕博、论文或训练经验 |
| AI 数据分析 / AI 产品分析 | 30-60K | 高 | 和数仓、指标体系、BI 背景结合好 |

样本中 AI 应用 / Agent / RAG 岗位的主流薪资集中在：

```text
20-40K
25-50K
30-60K
40-60K
```

粗略按岗位薪资中位数统计，样本月薪中位数约 `35K`，平均约 `34.6K`。
低于 20K 的多是初级、支持、实施、实习或偏工具使用岗位；
40K 以上通常要求 3-5 年以上经验、完整 AI Agent / RAG 项目经验或大厂业务经验。

## 6. 技术要求统计

公开样本里反复出现的技术关键词：

| 技术要求 | 出现频率 | 对当前路线的意义 |
|----------|----------|------------------|
| LLM / 大模型应用 | 很高 | 必须掌握调用、上下文、输出控制 |
| AI Agent | 很高 | 要会任务规划、工具调用、工作流 |
| RAG | 很高 | 当前项目 2 是核心竞争力 |
| Prompt Engineering | 高 | 要能讲清 prompt 版本、测试和边界 |
| Python | 高 | AI 应用和 RAG 后端常用 |
| Java / Go / 后端服务 | 中高 | 大厂 Agent 岗常要求工程后端 |
| LangChain / AutoGen / LlamaIndex | 中 | Agent / RAG 简历加分 |
| 向量数据库 / 检索系统 | 中高 | RAG 岗核心 |
| SQL / MySQL / Redis | 中高 | 数据问答和后端岗位常见 |
| 模型微调 / SFT / 评测 | 中 | 高阶岗位加分 |
| 推理部署 / GPU / vLLM | 中 | AI 平台和算法工程岗位重要 |
| 多模态 / OCR / CV | 中 | 偏产品方向或视觉方向 |

对你的当前背景，最应该突出的是：

```text
SQL + 数据仓库 + 指标口径 + RAG + NL2SQL + FastAPI + 评测 + 权限
```

这条线比硬冲“纯算法工程师”更匹配。

## 7. 岗位要求统计

| 要求 | 观察结论 |
|------|----------|
| 工作年限 | 主流是 3-5 年；1-3 年岗位存在但薪资多在 15-35K |

| 算法能力 | 高薪算法岗要求 NLP、多模态、推荐、搜索、训练、微调、评测 |
| 业务理解 | 财务 SaaS、电商、搜索推荐、办公 Agent、金融、教育、医疗都有场景要求 |

## 8. 公司规模观察

样本里公司规模大致分布：

| 公司规模 | 代表公司 | 岗位特征 |
|----------|----------|----------|
| 10000+ | 字节、美团、京东、百度、快手、蚂蚁 | 薪资高、流程规范、要求工程深度 |
| 1000-9999 | SenseTime、Shopee、同程、寒武纪、联想利泰 | 岗位稳定，AI 应用和平台岗较多 |
| 100-499 | 智源、深言、中科聚信、启天电子 | AI Agent / RAG 应用落地机会多 |
| 20-99 | 北京楷模、新智体、圜晖、研大教育 | 需求具体，薪资差异大 |
| 0-20 | 智映光年、清屿科技、东易天正、生橙视觉 | 机会前沿，但稳定性和管理成熟度要确认 |

投递建议：

- 大厂：优先投 AI 应用、RAG、数据分析 + AI、AI 平台，不要只投纯算法。
- 中小厂：可以投 AI Agent / RAG 应用落地，但要问清楚业务、现金流和技术负责人。
- 初创：薪资可能高，但要确认社保、试用期、加班、融资和产品真实进展。

## 12. 结论

从回龙观东大街出发，最合理的求职策略是：

1. 主投海淀北部和中部：上地、西二旗、学院路、中关村、西北旺。
2. 岗位主线放在 AI 应用、RAG、AI Agent 后端、数据问答、NL2SQL。
3. 薪资目标第一阶段可以看 `25-50K`，项目表达成熟后冲 `30-60K`。
4. 少投纯算法训练岗，多投“AI + 数据 + 工程落地”岗位。
5. 望京、酒仙桥、亦庄、马驹桥要把通勤成本算进选择，尤其亦庄和马驹桥不适合作为日常通勤首选。

最匹配你的方向不是“纯大模型算法工程师”，而是：

```text
RAG / NL2SQL / AI 数据问答 / AI 应用工程师
```

这类岗位既吃 AI 工程能力，也吃 SQL、指标口径、表结构和数据治理经验。
