## 面试主讲项目话术

我做了一个本地 RAG 知识问答 API 项目，用学习笔记和项目文档构建知识库。
离线侧负责文档清洗、chunk 切分、embedding 和索引构建；
在线侧负责用户问题检索、引用来源返回、拒答边界和 API 封装。

后续我又补了 query rewrite、固定测试集、拒答规则、权限标签、敏感信息控制、成本优化和演示稳定性检查。
这个项目重点不是追求模型多复杂，而是把一个 RAG 应用按生产链路拆成可解释、可评估、可排查的工程系统。
