# 金融信贷离线/实时仓库 + Agent 63 天学习计划

## 定位

这份计划把当前 AI 转型路线压缩并同步为 63 天。
Day 1-44 保留既有 Python、RAG、NL2SQL、服务化、测试和 Agent 基础成果。
从 Day 45 开始，学习主线统一改为“金融信贷离线/实时仓库 + Agent”。

目标不是单独学习传统数仓，也不是只做一个 Agent Demo，而是把离线指标、实时指标、实时特征、
指标口径、血缘追溯、权限合规和 Agent / RAG / NL2SQL 编排结合起来，形成可讲、可演示、
可继续扩展的信贷数据智能项目。

## Day 45-63 学习原则

从 Day 45 开始，每天必须同时包含：

- Agent 相关知识：意图识别、工具路由、前置条件、失败回退、评测、审计、解释、服务化；
- 数据仓库相关知识：离线分层、实时链路、指标口径、Schema Catalog、血缘、权限、质量、告警。

每天产出不能只是一段笔记，必须留下代码、文档、配置、测试集、报告或面试材料。

## 最终作品主线

推荐最终作品项目叫：

```text
金融信贷离线/实时仓库 Agent 数据问答系统
```

一句话介绍：

```text
基于离线仓库指标表、实时风险指标、指标口径知识库和 Agent 工具编排，
实现信贷经营指标查询、实时告警解释、口径问答、血缘追溯、日报生成和安全审计。
```

核心模块：

- 离线仓库模拟数据：ODS / DWD / DWS / ADS；
- 实时仓库模拟数据：事件流、实时指标、实时特征、告警记录；
- 指标字典、实时指标字典和 Schema Catalog；
- RAG 口径问答；
- NL2SQL 离线指标查询；
- 实时状态与告警查询工具；
- 血缘追溯工具；
- Agent 工具路由；
- SQL 校验、权限阻断和实时状态校验；
- 端到端评测；
- 审计记录和报告输出。

## 面试表达重点

面试时不要只说“我做了一个 Agent”，要说清楚：

- 数据底座同时包含离线仓库和实时仓库，不是随便查几张表；
- 离线仓库负责日报、历史分析、指标口径和训练样本；
- 实时仓库负责风控监控、反欺诈、实时特征、告警和分钟级状态；
- Agent 调 SQL 前必须理解指标口径、表层级、权限和分区；
- Agent 查实时指标时必须理解窗口、延迟、乱序、重复和告警等级；
- RAG 负责口径解释，NL2SQL 负责离线指标查询，实时工具负责实时状态，血缘工具负责追溯来源；
- SQL 校验负责只读、权限、敏感字段、时间范围和扫描成本；
- 端到端评测覆盖成功、拒答、补充条件、资料不足、系统异常、实时延迟和告警误报；
- 金融信贷场景必须重视隐私、审计、合规、实时错误和错误答案风险。

从 Day55 开始，Agent 学习需要从“手写流程 Demo”升级到“框架化编排 + 可观测评测”。

专项计划文档：

```text
docs/day55_plus_langchain_langgraph_langsmith_learning_plan.md
```

三者在本项目里的定位：

- LangChain：封装模型、Prompt、Tool、结构化输出和基础 Agent 调用；
- LangGraph：把意图识别、权限校验、工具路由、SQL 安全、实时延迟判断、人工审核和审计写成显式状态机；

这条补强线不替换原 Day55-63 主线，而是作为后续交付质量要求并入每一天的项目产出。
