## 今日任务拆解

### 任务 1：检查风险规则

重点规则：

- [x] `select *`：读取不必要字段
- [x] 缺少 `where`：可能全表扫描
- [x] 缺少 `dt`：数仓大表可能缺少分区过滤
- [x] `group by`：可能带来聚合开销
- [x] `order by`：可能带来全局排序成本
- [x] 有 `limit` 但无过滤：不代表扫描成本低

### 任务 2：优化输出

- 开发同学：知道哪里有风险、怎么改

## 面试表达

> 我做了一个 SQL 解释助手 CLI，用本地规则先模拟 AI 应用里的工具层。
