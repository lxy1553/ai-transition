## 今日核心知识点

### 1. RAG 是什么

RAG 全称是 Retrieval-Augmented Generation，中文叫“检索增强生成”。

大白话：

> 先从知识库里找相关资料，再让模型基于资料回答。

它解决的问题是：模型本身不知道你的项目文档、公司口径、表结构和最新规则，
所以需要先检索外部资料。

### 2. 为什么先整理资料

如果知识库资料质量差，后续再好的模型也容易答错。

例如：

- 文档过期，模型会基于旧口径回答
- 文档冲突，模型不知道信哪一个
- 文档太长，检索不到重点
- 文档没有来源，回答无法追溯

所以今天先做资料清单。

### 3. 第一批 RAG 项目范围

推荐范围：

```text
SQL 解释助手知识库
```

它包含：

- SQL 解释助手项目说明
- SQL 风险规则
- Tool Use 说明
- 结构化输出说明
- 项目讲解提纲
- 术语表

这个范围小、清晰、和你的数据仓库背景强相关。
