## 2. 数据流全景

### 2.1 实时数据流

```
行为埋点 ──▶ Kafka(topic: user_behavior) ──▶ Flink
                                                 │
                                                 ├─▶ 滑动窗口聚合(1h/24h/7d/30d)
                                                 │   · count_distinct(device_id)
                                                 │   · sum(operation_count) WHERE hour BETWEEN 22-06
                                                 │   · avg(session_duration)
                                                 │
                                                 ├─▶ JOIN 用户画像维表 (Caffeine 本地缓存)
                                                 │
                                                 ├─▶ 写入 Redis (在线特征, TTL=7d)
                                                 │    Key: feature:{user_id}:{feature_name}
                                                 │    Value: {value, ts}
                                                 │
                                                 └─▶ 写入 ClickHouse (实时监控)
                                                      Table: rt_feature_monitor
                                                      TTL: 90d
```

### 2.2 离线数据流

```
ODS (原始数据)
  │
  ▼
Spark ETL (T+1 凌晨执行)
  │
  ├─▶ DWD 清洗: 去重、格式标准化、敏感字段脱敏(tokenize)
  │
  ├─▶ DWS 特征宽表构建:
  │   · 基础画像: age_bucket, income_level, education, marital_status
  │   · 行为衍生: apply_cnt_7d/30d, night_ops_ratio_30d,
  │               avg_session_duration_7d, device_change_cnt_30d
  │   · 征信衍生: credit_score_normalized, debt_to_income_ratio,
  │               overdue_cnt_hist, query_cnt_3m
  │   · 多头衍生: multi_head_cnt_7d, multi_head_platform_type_cnt
  │   · 设备衍生: device_risk_score, device_rooted_flag,
  │               sim_change_cnt_30d
  │
  ├─▶ 特征PSI监控 (每日计算):
  │   psi = sum((actual_dist[i] - expected_dist[i]) * ln(actual_dist[i] / expected_dist[i]))
  │   告警阈值: PSI > 0.25
  │
  └─▶ ADS 样本构建:
      · Feast PIT Join (防止时间泄漏)
      · 左表: {user_id, application_time, label(DPD30+)}
      · 右表: DWS特征宽表 @ application_time 时间点
      · 输出: 训练样本 Parquet 文件
```

### 2.3 推理请求数据流

```
┌─────────────────────────────────────────────────────────────────┐
│  推理请求完整数据流 (单次请求)                                     │
│                                                                 │
│  1. 客户端发送: {user_id, product_type, apply_amount, device_id} │
│                                                                 │
│  2. API Gateway: 鉴权、限流、trace_id注入                        │
│                                                                 │
│  3. Decision Engine 并行获取 (asyncio.gather):                   │
│     ┌────────────────────────────────────────────────────────┐  │
│     │ FeatureService.get_online_features(user_id, features)   │  │
│     │   └─▶ Redis: HMGET feature:{user_id}:*                 │  │
│     │   超时: 50ms, 降级: 返回缓存值或默认值                    │  │
│     ├────────────────────────────────────────────────────────┤  │
│     │ CreditReportService.get_report(user_id)                │  │
│     │   └─▶ 第三方HTTP API / 本地缓存(TTL=24h)               │  │
│     │   超时: 200ms, 降级: 使用最近一次缓存报告                │  │
│     ├────────────────────────────────────────────────────────┤  │
│     │ MultiHeadService.query(user_id)                        │  │
│     │   └─▶ 第三方API / Redis缓存                             │  │
│     │   超时: 150ms, 降级: 标记为数据缺失，使用默认值           │  │
│     ├────────────────────────────────────────────────────────┤  │
│     │ DeviceFingerprintService.evaluate(device_id)           │  │
│     │   └─▶ Redis 查询设备风险画像                             │  │
│     │   超时: 30ms                                             │  │
│     └────────────────────────────────────────────────────────┘  │
│                                                                 │
│  4. 规则引擎执行 (~5ms):                                        │
│     · 黑名单检查 (Bloom Filter + Redis Set)                     │
│     · 硬拒绝规则 (年龄/地区/行业)                                │
│     · 反欺诈规则 (设备多账号关联)                                │
│                                                                 │
│  5. 模型推理 (~10ms):                                           │
│     · XGBoost/LightGBM predict(feature_vector)                 │
│     · SHAP TreeExplainer 计算特征贡献                          │
│     · 评分映射: score = base_score - factor * ln(odds)         │
│                                                                 │
│  6. 额度策略 (~1ms):                                            │
│     · 收入倍数法 / 分段定额法                                    │
│                                                                 │
│  7. 决策日志入Kafka (异步, fire-and-forget):                    │
│     topic: decision_log                                        │
│     {request_id, user_id, trace_id, features_snapshot,         │
│      model_name, model_version, score, decision,               │
│      credit_limit, shap_values, latency_ms, timestamp}         │
│                                                                 │
│  8. 返回响应: {decision, score, credit_limit, reason_codes}     │
└─────────────────────────────────────────────────────────────────┘
```

#### 4.2.2 特征平台：Feast 核心能力

```
Feast 解决的金融风控核心问题:

1. Point-in-Time Join (防止时间泄漏)
   问题: 训练样本 label 是 T+30 逾期标签，如果用 T+30 时的特征训练，
         模型会"看到未来"，上线后效果极差。
   方案: Feast.get_historical_features(
             entity_df=training_samples,    # {user_id, event_timestamp, label}
             features=feature_refs,
         )
         → Feast 自动取每个 event_timestamp 时刻的特征快照

2. 离线/在线特征一致性
   问题: 训练用离线特征(Spark批量算)，推理用在线特征(Redis查)，
         二者口径不一致会导致模型效果退化。
   方案: Feast 统一特征定义，离线从 Iceberg 取，在线从 Redis 取，
         保证 transformation 逻辑一致。

3. 特征血缘与回溯
   问题: 监管要求能解释"这个审批使用了哪些特征，值是多少"。
   方案: Feast FeatureView 记录特征定义和依赖，结合 Iceberg Time Travel
         可回溯任意历史时刻的特征快照。
```

### 5.2 规则引擎伪代码

```python
# decision_engine/rule_engine.py

from dataclasses import dataclass, field
from enum import Enum
from typing import Any, Callable, Optional
import operator
import yaml

class Decision(str, Enum):
    REJECT = "REJECT"
    APPROVE = "APPROVE"
    MANUAL_REVIEW = "MANUAL_REVIEW"
    REDUCE_LIMIT = "REDUCE_LIMIT"

@dataclass
class RuleResult:
    rule_id: str
    decision: Decision
    reason_code: str
    reason_desc: str
    triggered: bool = False

@dataclass
class Rule:
    id: str
    condition_expr: str
    decision: Decision
    reason_code: str
    reason_desc: str
    priority: int
    overridable: bool = True
    action: Optional[dict] = None

class RuleEngine:
    """
    轻量级规则引擎：按优先级分组执行，支持短路（hard_reject 命中即停止）。
    """
    
    # 安全的操作符白名单
    ALLOWED_OPS = {
        '>': operator.gt, '<': operator.lt,
        '>=': operator.ge, '<=': operator.le,
        '==': operator.eq, '!=': operator.ne,
        'and': lambda a, b: a and b,
        'or': lambda a, b: a or b,
        'in': lambda a, b: a in b,
    }
    
    def __init__(self, rules_config_path: str):
        self.rule_groups = self._load_rules(rules_config_path)
    
    def _load_rules(self, path: str) -> list[dict]:
        with open(path) as f:
            config = yaml.safe_load(f)
        return config['rule_groups']
    
    def evaluate(self, context: dict[str, Any]) -> list[RuleResult]:
        """
        按优先级执行规则组，hard_reject 组命中则短路。
        
        Args:
            context: 特征字典，如 {'age': 25, 'fraud_score': 0.3, ...}
        
        Returns:
            所有触发的规则结果列表
        """
        results: list[RuleResult] = []
        blacklist_set = context.get('_blacklist_set', set())
        context_with_sets = {**context, 'blacklist_set': blacklist_set}
        
        for group in sorted(self.rule_groups, key=lambda g: g['priority']):
            for rule_def in group['rules']:
                if self._evaluate_condition(rule_def['condition'], context_with_sets):
                    result = RuleResult(
                        rule_id=rule_def['id'],
                        decision=Decision(rule_def['decision']),
                        reason_code=rule_def['reason_code'],
                        reason_desc=rule_def['reason_desc'],
                        triggered=True,
                    )
                    results.append(result)
                    
                    # hard_reject: 短路，不再执行后续规则
                    if group['name'] == 'hard_reject' and \
                       not rule_def.get('overridable', True):
                        return results
        
        # 如果没有拒绝，默认通过
        if not any(r.decision == Decision.REJECT for r in results):
            results.append(RuleResult(
                rule_id="DEFAULT_APPROVE",
                decision=Decision.APPROVE,
                reason_code="RC_OK",
                reason_desc="通过所有规则检查",
            ))
        
        return results
    
    def _evaluate_condition(self, condition: str, context: dict) -> bool:
        """
        安全地评估条件表达式。
        实际生产中使用 ast.literal_eval 或构建表达式 AST 遍历，
        这里用简化版本演示逻辑。
        """
        # 生产环境使用受限的表达式解析器，例如：
        # from simpleeval import simple_eval
        # return simple_eval(condition, names=context)
        return self._safe_eval(condition, context)
    
    def _safe_eval(self, expr: str, context: dict) -> bool:
        """使用Python AST的安全表达式求值器"""
        import ast
        
        class SafeEval(ast.NodeVisitor):
            def __init__(self, context):
                self.context = context
                self.result = None
            
            def visit_Compare(self, node):
                left = self._get_value(node.left)
                for op_node, comparator in zip(node.ops, node.comparators):
                    right = self._get_value(comparator)
                    op_func = {
                        ast.Gt: operator.gt, ast.Lt: operator.lt,
                        ast.GtE: operator.ge, ast.LtE: operator.le,
                        ast.Eq: operator.eq, ast.NotEq: operator.ne,
                    }[type(op_node)]
                    if not op_func(left, right):
                        self.result = False
                        return
                self.result = True
            
            def visit_BoolOp(self, node):
                if isinstance(node.op, ast.And):
                    self.result = all(self._get_value(v) for v in node.values)
                elif isinstance(node.op, ast.Or):
                    self.result = any(self._get_value(v) for v in node.values)
            
            def _get_value(self, node):
                if isinstance(node, ast.Constant):
                    return node.value
                elif isinstance(node, ast.Name):
                    return self.context[node.id]
                elif isinstance(node, ast.Compare):
                    self.visit_Compare(node)
                    return self.result
                raise ValueError(f"Unsupported node: {type(node)}")
        
        tree = ast.parse(expr, mode='eval')
        evaluator = SafeEval(context)
        evaluator.visit(tree.body)
        return evaluator.result
```

### 5.3 推理引擎核心（决策流水线）

```python
# decision_engine/inference_pipeline.py

import asyncio
import time
import hashlib
from dataclasses import dataclass, field
from typing import Optional
import numpy as np
import xgboost as xgb
import shap
from aiokafka import AIOKafkaProducer

# ── 数据结构 ──────────────────────────────────────────────

@dataclass
class InferenceRequest:
    user_id: str
    product_type: str
    apply_amount: float
    device_id: str
    trace_id: str

@dataclass
class FeatureSnapshot:
    """推理时的特征快照（完整记录，用于日志和回溯）"""
    user_id: str
    features: dict[str, float]
    feature_version: str
    missing_features: list[str]
    degraded_features: list[str]  # 降级获取的特征

@dataclass
class DecisionResult:
    request_id: str
    user_id: str
    trace_id: str
    decision: str              # APPROVE / REJECT / MANUAL_REVIEW
    score: float               # 信用评分 (300-900)
    default_prob: float        # 违约概率
    credit_limit: float        # 授信额度
    model_name: str
    model_version: str
    reason_codes: list[str]
    shap_values: dict[str, float]  # Top-N 特征贡献
    latency_ms: float
    feature_snapshot: FeatureSnapshot

# ── 降级策略 ──────────────────────────────────────────────

class DegradationPolicy:
    """超时降级策略"""
    
    FEATURE_DEFAULTS = {
        'multi_head_cnt_7d': 0,
        'multi_head_cnt_30d': 0,
        'debt_to_income_ratio': 0.5,
        'credit_score_normalized': 0.5,
    }
    
    @classmethod
    def get_default(cls, feature_name: str) -> float:
        return cls.FEATURE_DEFAULTS.get(feature_name, 0.0)

# ── 推理流水线 ────────────────────────────────────────────

class InferencePipeline:
    """
    风控推理流水线：规则检查 → 模型打分 → 额度策略
    
    设计原则:
    - 每个阶段可独立超时和降级
    - 所有IO操作并行(asyncio.gather)
    - 决策日志异步写Kafka(不阻塞响应)
    """
    
    def __init__(
        self,
        rule_engine: 'RuleEngine',
        model_registry: 'ModelRegistry',
        feature_service: 'FeatureService',
        credit_report_service: 'CreditReportService',
        kafka_producer: AIOKafkaProducer,
    ):
        self.rule_engine = rule_engine
        self.model_registry = model_registry
        self.feature_service = feature_service
        self.credit_report_service = credit_report_service
        self.kafka_producer = kafka_producer
    
    async def execute(self, request: InferenceRequest) -> DecisionResult:
        t_start = time.perf_counter()
        request_id = self._gen_request_id(request)
        
        # ── Phase 1: 并行获取所有特征和数据 ──
        feature_snapshot, credit_data, device_data, multi_head_data = \
            await self._gather_features(request)
        
        # 构建完整的特征上下文
        context = {
            **feature_snapshot.features,
            **credit_data,
            **device_data,
            **multi_head_data,
            '_blacklist_set': await self._load_blacklist(),
        }
        
        # ── Phase 2: 硬规则检查（同步，低延迟）──
        rule_results = self.rule_engine.evaluate(context)
        reject_rules = [r for r in rule_results if r.decision == 'REJECT']
        
        if reject_rules:
            latency_ms = (time.perf_counter() - t_start) * 1000
            result = DecisionResult(
                request_id=request_id,
                user_id=request.user_id,
                trace_id=request.trace_id,
                decision='REJECT',
                score=0,
                default_prob=1.0,
                credit_limit=0,
                model_name='rule_engine',
                model_version='N/A',
                reason_codes=[r.reason_code for r in reject_rules],
                shap_values={},
                latency_ms=latency_ms,
                feature_snapshot=feature_snapshot,
            )
            await self._log_decision(result)
            return result
        
        # ── Phase 3: A卡模型打分 ──
        model = self.model_registry.get_active_model('A_card')
        feature_vector = self._build_feature_vector(context, model.feature_names)
        
        # XGBoost predict
        dmatrix = xgb.DMatrix(np.array([feature_vector]), feature_names=model.feature_names)
        default_prob = float(model.booster.predict(dmatrix)[0])
        
        # SHAP 解释
        explainer = model.shap_explainer  # 预加载的 TreeExplainer
        shap_vals = explainer.shap_values(dmatrix)[0]
        shap_contributions = dict(
            sorted(
                zip(model.feature_names, shap_vals),
                key=lambda x: abs(x[1]), reverse=True
            )[:10]  # Top-10 贡献特征
        )
        
        # 评分卡映射
        score = self._prob_to_score(default_prob)
        
        # ── Phase 4: 额度策略 ──
        credit_limit = self._calculate_limit(
            score=score,
            monthly_income=context.get('monthly_income', 0),
            debt_ratio=context.get('debt_to_income_ratio', 0.5),
            apply_amount=request.apply_amount,
        )
        
        # ── Phase 5: 组装结果 ──
        latency_ms = (time.perf_counter() - t_start) * 1000
        
        decision = 'APPROVE' if score >= 500 else 'MANUAL_REVIEW'
        
        result = DecisionResult(
            request_id=request_id,
            user_id=request.user_id,
            trace_id=request.trace_id,
            decision=decision,
            score=score,
            default_prob=default_prob,
            credit_limit=credit_limit,
            model_name=model.name,
            model_version=model.version,
            reason_codes=[r.reason_code for r in rule_results if r.triggered],
            shap_values=shap_contributions,
            latency_ms=latency_ms,
            feature_snapshot=feature_snapshot,
        )
        
        # 异步写入决策日志（不阻塞响应）
        asyncio.create_task(self._log_decision(result))
        
        return result
    
    async def _gather_features(
        self, request: InferenceRequest
    ) -> tuple[FeatureSnapshot, dict, dict, dict]:
        """
        并行获取所有数据源，每个数据源有独立超时和降级策略。
        """
        results = await asyncio.gather(
            self._fetch_features_with_fallback(request),
            self._fetch_credit_report_with_fallback(request.user_id),
            self._fetch_device_data_with_fallback(request.device_id),
            self._fetch_multi_head_with_fallback(request.user_id),
            return_exceptions=True,
        )
        
        # 解包结果，异常时使用默认值
        feature_snapshot = results[0] if not isinstance(results[0], Exception) \
            else FeatureSnapshot(user_id=request.user_id, features={}, ...)
        credit_data = results[1] if not isinstance(results[1], Exception) else {}
        device_data = results[2] if not isinstance(results[2], Exception) else {}
        multi_head_data = results[3] if not isinstance(results[3], Exception) else {}
        
        return feature_snapshot, credit_data, device_data, multi_head_data
    
    async def _fetch_features_with_fallback(
        self, request: InferenceRequest, timeout: float = 0.05
    ) -> FeatureSnapshot:
        """获取在线特征，50ms超时降级"""
        try:
            return await asyncio.wait_for(
                self.feature_service.get_online_features(request.user_id),
                timeout=timeout,
            )
        except asyncio.TimeoutError:
            # 降级：返回缓存值或默认值
            degraded = self.feature_service.get_cached_features(request.user_id)
            degraded.missing_features = [
                f for f in self.feature_service.FEATURE_LIST
                if f not in degraded.features
            ]
            return degraded
    
    def _prob_to_score(self, prob: float) -> float:
        """违约概率 → 信用评分 (300-900)"""
        base_score = 600
        pdo = 50  # Points to Double Odds
        factor = pdo / np.log(2)
        odds = (1 - prob) / prob
        score = base_score + factor * np.log(odds)
        return max(300, min(900, score))
    
    def _calculate_limit(
        self, score: float, monthly_income: float,
        debt_ratio: float, apply_amount: float
    ) -> float:
        """额度计算策略"""
        if monthly_income <= 0:
            return 0
        
        # 基础额度：月收入的倍数
        if score >= 700:
            income_multiplier = 18
        elif score >= 600:
            income_multiplier = 12
        elif score >= 500:
            income_multiplier = 6
        else:
            income_multiplier = 0
        
        base_limit = monthly_income * income_multiplier
        
        # 负债率调整
        debt_adjustment = max(0.3, 1.0 - debt_ratio)
        
        # 不超过申请金额
        final_limit = min(base_limit * debt_adjustment, apply_amount)
        
        return round(final_limit, 2)
    
    def _build_feature_vector(
        self, context: dict, feature_names: list[str]
    ) -> np.ndarray:
        """按模型要求的特征顺序构建向量"""
        vec = []
        for name in feature_names:
            val = context.get(name)
            if val is None:
                val = DegradationPolicy.get_default(name)
            vec.append(float(val))
        return np.array(vec, dtype=np.float32)
    
    async def _log_decision(self, result: DecisionResult):
        """异步写决策日志到 Kafka"""
        log_entry = {
            'request_id': result.request_id,
            'user_id': result.user_id,
            'trace_id': result.trace_id,
            'decision': result.decision,
            'score': result.score,
            'default_prob': result.default_prob,
            'credit_limit': result.credit_limit,
            'model_name': result.model_name,
            'model_version': result.model_version,
            'reason_codes': result.reason_codes,
            'shap_top10': result.shap_values,
            'latency_ms': result.latency_ms,
            'feature_snapshot': result.feature_snapshot.features,
            'feature_version': result.feature_snapshot.feature_version,
            'missing_features': result.feature_snapshot.missing_features,
            'timestamp': time.time(),
        }
        await self.kafka_producer.send_and_wait('decision_log', log_entry)
    
    def _gen_request_id(self, request: InferenceRequest) -> str:
        raw = f"{request.user_id}:{request.trace_id}:{time.time()}"
        return hashlib.sha256(raw.encode()).hexdigest()[:16]
```

```python
# ── 2. 特征工程: WOE/IV ────────────────────────────────

def calculate_woe_iv(df, feature: str, target: str = 'label') -> dict:
    """
    计算单个特征的 WOE 和 IV 值（评分卡标准方法）。
    
    WOE_i = ln(Distribution_Good_i / Distribution_Bad_i)
    IV = Σ (Distr_Good_i - Distr_Bad_i) * WOE_i
    
    IV 解释:
    - < 0.02: 无预测能力
    - 0.02 - 0.1: 弱
    - 0.1 - 0.3: 中等
    - 0.3 - 0.5: 强
    - > 0.5: 可疑（可能过拟合/时间泄漏）
    """
    total_good = (df[target] == 0).sum()
    total_bad = (df[target] == 1).sum()
    
    if df[feature].dtype in ('object', 'category'):
        grouped = df.groupby(feature)[target].agg(['sum', 'count'])
    else:
        df['_bin'] = pd.qcut(df[feature], q=10, duplicates='drop')
        grouped = df.groupby('_bin')[target].agg(['sum', 'count'])
    
    result = []
    total_iv = 0
    
    for bin_name, row in grouped.iterrows():
        bad = row['sum']
        good = row['count'] - bad
        
        dist_good = good / total_good if total_good > 0 else 0
        dist_bad = bad / total_bad if total_bad > 0 else 0
        
        # WOE 平滑处理（防止 ln(0)）
        dist_good = max(dist_good, 1e-6)
        dist_bad = max(dist_bad, 1e-6)
        
        woe = np.log(dist_good / dist_bad)
        iv = (dist_good - dist_bad) * woe
        
        result.append({
            'feature': feature,
            'bin': str(bin_name),
            'good': good, 'bad': bad,
            'woe': round(woe, 4),
            'iv': round(iv, 4),
        })
        total_iv += iv
    
    return {'feature': feature, 'total_iv': round(total_iv, 4), 'bins': result}

# ── 4. 评估函数 ──────────────────────────────────────

def _calculate_ks(y_true, y_pred_proba):
    """KS = max(|TPR - FPR|)"""
    sorted_idx = np.argsort(y_pred_proba)[::-1]
    y_true_sorted = y_true[sorted_idx]
    
    n_pos = np.sum(y_true == 1)
    n_neg = np.sum(y_true == 0)
    
    tpr = np.cumsum(y_true_sorted == 1) / n_pos
    fpr = np.cumsum(y_true_sorted == 0) / n_neg
    
    return np.max(np.abs(tpr - fpr))


def _calculate_psi(expected_scores, actual_scores, bins=10):
    """
    PSI = Σ (actual% - expected%) * ln(actual% / expected%)
    监测分数分布偏移。PSI > 0.25 表示显著偏移，模型可能需要重训。
    """
    expected_percents = np.histogram(expected_scores, bins=bins)[0] / len(expected_scores)
    actual_percents = np.histogram(actual_scores, bins=bins)[0] / len(actual_scores)
    
    expected_percents = np.clip(expected_percents, 1e-6, None)
    actual_percents = np.clip(actual_percents, 1e-6, None)
    
    return np.sum((actual_percents - expected_percents) * 
                  np.log(actual_percents / expected_percents))


def _calculate_lift(y_true, y_pred_proba, bins=10):
    """Lift = 每个分箱的坏账率 / 整体坏账率"""
    df = pd.DataFrame({'true': y_true, 'score': y_pred_proba})
    df['bin'] = pd.qcut(df['score'], q=bins, labels=False, duplicates='drop')
    
    overall_bad_rate = y_true.mean()
    
    lift_result = []
    for b in sorted(df['bin'].unique()):
        bin_df = df[df['bin'] == b]
        bin_bad_rate = bin_df['true'].mean()
        lift = bin_bad_rate / overall_bad_rate if overall_bad_rate > 0 else 0
        lift_result.append({

            'bad_rate': round(bin_bad_rate, 4),
            'lift': round(lift, 4),
        })
    
    return {'overall_bad_rate': round(overall_bad_rate, 4), 'bins': lift_result}
```

```python
# monitoring/feature_monitor.py

"""
特征漂移监控服务

每日运行，对比当前特征分布 vs 基准分布（训练集），
计算 PSI，超过阈值告警。
"""

import json
from datetime import datetime, timedelta
import numpy as np
import pandas as pd
from pyspark.sql import SparkSession
import requests  # 告警通知


class FeatureDriftMonitor:
    """
    特征漂移监控器
    
    监控指标:
    - PSI (Population Stability Index): 特征分布偏移度
    - 空值率变化
    - 特征均值/方差漂移
    """
    
    PSI_THRESHOLD_WARN = 0.1    # 黄色告警
    PSI_THRESHOLD_CRITICAL = 0.25  # 红色告警(可能需触发熔断)
    
    def __init__(self, feast_repo_path: str, webhook_url: str):
        self.feast = FeatureStore(repo_path=feast_repo_path)
        self.spark = SparkSession.builder.getOrCreate()
        self.webhook_url = webhook_url
        self.baseline = self._load_baseline_distribution()
    
    def _load_baseline_distribution(self) -> dict:
        """加载训练集特征分布作为基准"""
        # 从 MLflow 中获取训练时的特征分布统计
        import mlflow
        client = mlflow.tracking.MlflowClient()
        prod_model = client.get_latest_versions(
            "credit_a_card_xgb", stages=["Production"]
        )[0]
        run = client.get_run(prod_model.run_id)
        baseline_path = run.data.tags.get('feature_distribution_path')
        
        with open(baseline_path) as f:
            return json.load(f)
    
    def run_daily_check(self, check_date: str = None):
        """每日特征漂移检查"""
        if check_date is None:
            check_date = (datetime.now() - timedelta(days=1)).strftime('%Y-%m-%d')
        
        # 获取当日特征分布
        current_dist = self._get_current_distribution(check_date)
        
        alerts = []
        psi_results = {}
        
        for feature, baseline_stats in self.baseline.items():
            if feature not in current_dist:
                continue
            
            current_stats = current_dist[feature]
            
            # 计算 PSI
            psi = self._calculate_psi(
                baseline_stats['histogram'],
                current_stats['histogram'],
            )
            psi_results[feature] = psi
            
            # 评估告警级别
            if psi > self.PSI_THRESHOLD_CRITICAL:
                alerts.append({
                    'feature': feature,
                    'psi': round(psi, 4),
                    'level': 'CRITICAL',
                    'message': f'特征 {feature} PSI={psi:.4f} 超过严重阈值 0.25',
                })
            elif psi > self.PSI_THRESHOLD_WARN:
                alerts.append({
                    'feature': feature,
                    'psi': round(psi, 4),
                    'level': 'WARNING',
                    'message': f'特征 {feature} PSI={psi:.4f} 超过告警阈值 0.1',
                })
            
            # 检查空值率变化
            null_rate_change = abs(
                current_stats.get('null_rate', 0) - 
                baseline_stats.get('null_rate', 0)
            )
            if null_rate_change > 0.3:
                alerts.append({
                    'feature': feature,
                    'null_rate_change': round(null_rate_change, 4),
                    'level': 'WARNING',
                    'message': f'特征 {feature} 空值率变化 {null_rate_change:.2%}',
                })
        
        # 生成报告
        report = {
            'check_date': check_date,
            'total_features': len(psi_results),
            'features_above_025': sum(1 for p in psi_results.values() if p > 0.25),
            'features_above_01': sum(1 for p in psi_results.values() if p > 0.1),
            'alerts': alerts,
            'psi_details': psi_results,
        }
        
        # 发送告警
        if alerts:
            self._send_alert(report)
        
        return report
    
    def _get_current_distribution(self, date: str) -> dict:
        """获取指定日期的特征分布"""
        query = f"""
            SELECT * FROM dws.user_risk_feature_wide
            WHERE dt = '{date}'
        """
        df = self.spark.sql(query).toPandas()
        
        distributions = {}
        for col in df.select_dtypes(include=[np.number]).columns:
            if col in ('user_id', 'application_id'):
                continue
            
            # 计算直方图和空值率
            hist, _ = np.histogram(df[col].dropna(), bins=10)
            distributions[col] = {
                'histogram': hist.tolist(),
                'null_rate': float(df[col].isna().mean()),
                'mean': float(df[col].mean()),
                'std': float(df[col].std()),
            }
        
        return distributions
    
    def _calculate_psi(self, expected_hist: list, actual_hist: list) -> float:
        """计算 PSI"""
        expected = np.array(expected_hist, dtype=float)
        actual = np.array(actual_hist, dtype=float)
        
        expected_pct = expected / expected.sum()
        actual_pct = actual / actual.sum()
        
        # 防止除零
        expected_pct = np.clip(expected_pct, 1e-6, None)
        actual_pct = np.clip(actual_pct, 1e-6, None)
        
        return float(np.sum((actual_pct - expected_pct) * 
                             np.log(actual_pct / expected_pct)))
    
    def _send_alert(self, report: dict):
        """发送告警通知"""
        criticals = [a for a in report['alerts'] if a['level'] == 'CRITICAL']
        warnings = [a for a in report['alerts'] if a['level'] == 'WARNING']
        
        message = f"""
        ╔══════════════════════════════════════════════╗
        ║  特征 PSI 监控日报 - {report['check_date']}        ║
        ╠══════════════════════════════════════════════╣
        ║  监控特征总数: {report['total_features']}                          ║
        ║  PSI > 0.25 (严重): {report['features_above_025']}                          ║
        ║  PSI > 0.10 (告警): {report['features_above_01']}                          ║
        ╠══════════════════════════════════════════════╣
        """
        
        if criticals:
            message += "║  🔴 CRITICAL:                               ║\n"
            for c in criticals:
                message += f"║    {c['feature']}: PSI={c['psi']:.4f}           ║\n"
        
        if warnings:
            message += "║  🟡 WARNING:                                 ║\n"
            for w in warnings:
                message += f"║    {w['feature']}: PSI={w['psi']:.4f}           ║\n"
        
        message += "╚══════════════════════════════════════════════╝"
        
        # 发送到钉钉/飞书/企业微信
        requests.post(self.webhook_url, json={
            'msgtype': 'text',
            'text': {'content': message},
        })


# ── 模型性能熔断 ──────────────────────────────────────

class ModelCircuitBreaker:
    """
    模型熔断器
    
    监控线上逾期率，如果突增超过阈值，自动切换至备用模型或纯规则模式。
    
    状态机: CLOSED → OPEN → HALF_OPEN → CLOSED
    """
    
    def __init__(self, prometheus_url: str):
        self.prometheus_url = prometheus_url
        self.state = 'CLOSED'  # CLOSED / OPEN / HALF_OPEN
        self.failure_window = 300  # 5分钟观察窗口
    
    def check(self) -> str:
        """
        检查线上逾期率是否异常。
        
        指标来源: Prometheus
        - rate(reject_rate[5m]): 拒绝率突降（可能漏过坏人）
        - 贷后 T+15 早期逾期率: 最敏感的预警信号
        """
        # 查询 Prometheus 指标
        query = '''
            (avg_over_time(early_delinquency_rate_t15[1h]) - 
             avg_over_time(early_delinquency_rate_t15[24h])) 
            / avg_over_time(early_delinquency_rate_t15[24h])
        '''
        
        resp = requests.get(
            f"{self.prometheus_url}/api/v1/query",
            params={'query': query}
        )
        
        if resp.status_code == 200:
            result = resp.json()
            if result['data']['result']:
                change_ratio = float(result['data']['result'][0]['value'][1])
                
                if change_ratio > 0.3:  # 逾期率突增30%
                    self.state = 'OPEN'
                    self._trigger_fallback()
                    return 'OPEN'
        
        return self.state
    
    def _trigger_fallback(self):
        """触发降级：切换至备用模型或纯规则模式"""
        # 1. 更新 MLflow Model Registry → 降级到 Staging 备用模型
        # 2. 或切换至纯规则引擎模式（拒绝率可能上升，但风险可控）
        # 3. 发送 P0 告警
        
        requests.post(self.webhook_url, json={
            'msgtype': 'text',
            'text': {
                'content': '🔴 [P0] 模型熔断触发！逾期率突增 >30%。'
                           '已自动切换至备用模型。请立即排查原因。'
            },
        })
```

```python
"""
A/B 测试流量路由

支持基于 user_id 哈希的稳定分流。
流量分配策略:
- Champion (Production): 80%
- Challenger 1 (Staging): 15%
- Challenger 2 (Staging): 5%

import hashlib

class ABTrafficRouter:
    """
    冠军/挑战者 A/B 测试路由器
    
    特点:
    - 基于 user_id MD5 哈希分流，同一用户始终路由到同一模型
    - 支持分层抽样（按城市/收入分层）
    - 分流比例热配（配置中心下发）
    """
    
    def __init__(self, config_service):
        self.config = config_service.get('ab_test_config')
        # config = {
        #     'enabled': True,
        #     'buckets': [
        #         {'name': 'champion_v3', 'model': 'credit_a_card_xgb', 
        #          'version': '3', 'weight': 80},
        #         {'name': 'challenger_v4', 'model': 'credit_a_card_xgb', 
        #          'version': '4', 'weight': 15},
        #         {'name': 'challenger_lgb', 'model': 'credit_a_card_lgb', 
        #          'version': '1', 'weight': 5},
        #     ]
        # }
    
    def route(self, user_id: str) -> dict:
        """
        根据 user_id 哈希确定路由目标。
        
        Hash 范围: [0, 9999]
        各分桶按权重分配区间。
        """
        if not self.config.get('enabled', False):
            # A/B 测试未开启，全部走冠军模型
            return self.config['buckets'][0]
        
        hash_val = int(hashlib.md5(user_id.encode()).hexdigest(), 16) % 10000
        
        cumulative = 0
        for bucket in self.config['buckets']:
            # weight 是百分制，如 80 → 对应 8000/10000
            bucket_range = int(bucket['weight'] * 100)
            if hash_val < cumulative + bucket_range:
                return bucket
            cumulative += bucket_range
        
        # fallback to champion
        return self.config['buckets'][0]
```
